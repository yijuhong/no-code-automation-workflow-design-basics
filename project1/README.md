# 프로젝트 1 기획서: CS 고객 문의 분류 자동화 도구 비교

## 1. 프로젝트 개요

이 프로젝트는 CS 담당자가 고객 문의를 내부 접수 폼에 입력하면, 자동화 도구가 문의의 우선순위에 따라 처리 흐름을 나누고 후속 기록과 알림을 실행하는 워크플로우를 설계한다.

동일한 워크플로우를 Make와 Zapier로 각각 구현할 수 있도록 구조를 통일하고, 두 도구의 설정 방식과 사용 경험을 비교하는 것을 목표로 한다.

## 2. 업무 시나리오

CS 담당자는 이메일, 채팅, 전화 등으로 들어온 고객 문의를 확인한 뒤 내부 Google Form에 문의 내용을 표준화해 입력한다. 입력 데이터는 실제 고객정보가 아닌 가상 고객 문의 데이터로 구성한다.

CS 담당자는 문의의 심각도와 고객 영향도를 기준으로 우선순위 점수를 0-100점 사이로 입력한다. 자동화 워크플로우는 이 점수를 기준으로 긴급 문의와 일반 문의를 구분한다.

## 3. 자동화 주제

**주제:** CS 고객 문의 우선순위 분류 및 알림 자동화

**목적:** 반복적으로 접수되는 고객 문의를 우선순위에 따라 자동 분류하고, 처리 대상 시트와 Slack 알림을 분기해 CS 담당자의 초기 분류 업무를 줄인다.

**비교 도구:** Make, Zapier

**알림 채널:** Slack `#automation-test`

## 4. 입력 데이터

Google Form은 CS 담당자가 사용하는 내부 접수 폼으로 설정한다.

| 필드 | 설명 |
| --- | --- |
| 고객명 | 가상 고객 이름 |
| 이메일 | 테스트용 이메일 |
| 문의 유형 | 결제, 기술지원, 계정, 일반 문의 등 |
| 문의 내용 | 고객 문의 요약 |
| 우선순위 점수 | CS 담당자가 판단한 0-100점 숫자 |

## 5. 워크플로우 구조

두 도구 모두 동일한 흐름으로 설계한다.

1. CS 담당자가 Google Form에 고객 문의를 입력한다.
2. Google Form 응답이 Google Sheets에 새 행으로 저장된다.
3. 자동화 도구가 Google Sheets의 새 행을 감지한다.
4. 우선순위 점수를 기준으로 조건 분기한다.
5. 분기 결과에 따라 Google Sheets에 기록하고 Slack 알림을 전송한다.

## 6. 조건 분기

| 분기 | 조건 | 처리 의미 |
| --- | --- | --- |
| 긴급 문의 | 우선순위 점수 80점 이상 | 빠른 확인과 우선 대응이 필요한 문의 |
| 일반 문의 | 우선순위 점수 80점 미만 | 일반 처리 흐름으로 접수할 문의 |

## 7. 분기별 처리 흐름

### 긴급 문의

- Google Sheets의 `긴급 문의` 탭에 문의 정보를 추가한다.
- Slack `#automation-test` 채널에 긴급 문의 알림을 보낸다.

### 일반 문의

- Google Sheets의 `일반 문의` 탭에 문의 정보를 추가한다.
- Slack `#automation-test` 채널에 일반 문의 접수 알림을 보낸다.

## 8. Make 구현 방향

Make에서는 Google Sheets의 새 행을 감지하는 모듈을 Trigger로 사용한다. 이후 Router 또는 Filter를 사용해 우선순위 점수 기준으로 긴급 문의와 일반 문의 경로를 나눈다.

각 경로에는 Google Sheets 행 추가 모듈과 Slack 메시지 전송 모듈을 배치한다. 시각적 캔버스에서 분기 구조가 명확히 보이기 때문에 워크플로우 전체 흐름을 설명하기 쉽다.

## 9. Zapier 구현 방향

Zapier에서는 Google Sheets의 New Spreadsheet Row 이벤트를 Trigger로 사용한다. 이후 Paths를 사용해 우선순위 점수 기준으로 긴급 문의와 일반 문의 경로를 나눈다.

각 Path에는 Google Sheets 행 추가 Action과 Slack 메시지 전송 Action을 배치한다. Zapier는 단계형 설정 방식이므로 각 단계의 조건과 실행 결과를 순서대로 확인하는 방식으로 구현한다.

## 10. 비교 관점

비교 분석 보고서는 과제 평가 기준에 맞춰 두 도구의 차이를 근거 중심으로 정리한다.

| 비교 항목 | 비교 의도 |
| --- | --- |
| UI/UX | 시각적 캔버스 방식과 단계형 설정 방식의 차이 |
| 설정 난이도 | Trigger, 조건 분기, Action 설정 과정의 직관성 |
| 조건 분기 방식 | Make Router/Filter와 Zapier Paths의 표현력 |
| 연동 서비스 | Google Sheets와 Slack 연동 설정의 편의성 |
| 무료 또는 trial 범위 | 실습에 필요한 기능을 비용 부담 없이 사용할 수 있는지 |
| 실행 로그 | 각 단계의 입력값, 출력값, 오류를 확인하는 방식 |
| 유지보수성 | 추후 분기나 Action을 추가할 때의 확장성 |

## 12. 최종 결정사항

이 프로젝트는 CS 담당자가 입력한 가상 고객 문의를 기준으로 Make와 Zapier의 조건 분기 자동화 구현 방식을 비교한다. 자동화 흐름은 Google Form, Google Sheets, Slack을 공통 구성 요소로 사용하며, 우선순위 점수 80점을 기준으로 긴급 문의와 일반 문의를 구분한다.
